Get Started

Welcome to the Faircorp Smart Building API documentation! This API allows you to manage a smart building and its entities, including rooms, heaters, and windows. It also enables the administrator to manage users and assign roles and access to them in the system.

To use the API, you will need to make HTTP requests to the appropriate endpoints with the correct request type and any necessary parameters or body data. You can find a list of the available endpoints and their request types in the "API Reference" section below.

Prerequisites

To use the API, you will need the following::

  • A computer with internet access
  • A REST client (such as Postman or Insomnia)
  • The API endpoint URL

Authenticating

The API uses HTTP basic authentication to authenticate users. You will need to provide a valid username and password in the Authorization header of each request.

API Reference

The API has the following endpoints:

Buildings:
GET /api/buildings retrieves a list of all buildings
GET /api/buildings/{id} retrieves information about a specific building
POST /api/buildings creates a new building
PUT /api/buildings/{id}/switchHeaters updates the status of heaters in a specific building
PUT /api/buildings/{id}/switchWindows updates the status of windows in a specific building
DELETE /api/buildings/{id} deletes a specific building
Rooms:
GET /api/rooms retrieves a list of all buildings
GET /api/rooms/{id} retrieves information about a specific building
POST /api/rooms creates a new building
PUT /api/rooms/{id}/switchHeaters updates the status of heaters in a specific room
PUT /api/rooms/{id}/switchWindows updates the status of windows in a specific room
DELETE /api/buildings/{id} deletes a specific building
Windows:
GET /api/windows retrieves a list of all windows
GET /api/windows/{id} retrieves information about a specific window
POST /api/windows creates a new window
PUT /api/windows/{id}/switch updates the status of a specific window
DELETE /api/windows/{id} deletes a specific window
Heaters:
GET /api/heaters retrieves a list of all heaters
GET /api/heaters/{id} retrieves information about a specific heater
POST /api/heaters creates a new heater
PUT /api/heaters/{id}/switch updates the status of a specific heater
DELETE /api/heaters/{id} updates the status of a specific heater
Users:
GET /api/users retrieves a list of all users
GET /api/users/{username} retrieves information about a specific user
POST /api/users creates a new user
DELETE /api/users/{username} deletes a specific user

Request and Response Format

The API uses JSON format for both request and response bodies.

Error Handling

If an error occurs, the API will return an HTTP status code indicating the type of error. The response body will also include an error message.

Example

Here is an example of how to create a new building using the POST /api/buildings

  1. Open your REST client
  2. Set the request method to POST
    3.               
                $ faircorp  ——POST
                  {
                    "id": 1,
                    "name": "Building 1",
                    "floors": [
                      {
                        "id": 1,
                        "name": "Floor 1",
                        "rooms": [
                          {
                            "id": 1,
                            "name": "Room 1",
                            "currentTemperature": 20.0,
                            "targetTemperature": 20.0,
                            "heaters": [
                              {
                                "id": 1,
                                "name": "Heater 1",
                                "status": "ON"
                              }
                            ],
                            "windows": [
                              {
                                "id": 1,
                                "name": "Window 1",
                                "status": "OPEN"
                              }
                            ]
                          }
  3. Set the request URL to the API endpoint followed by /api/buildings/{id}, replacing {id} with the ID of the building you want to retrieve
  4. Set the Authorization header to include your username and password
  5. Send the request

If the request is successful, you will receive a response with a status code of 200 OK and a JSON object containing information about the building. If an error occurs, you will receive an appropriate status code and an error message.
